---
title: Encounter Resource
description: Clinical encounters, visits, and patient interactions
---

The Encounter resource represents an interaction between a patient and healthcare provider(s). All encounters must reference a valid Patient.

## Endpoints

### Create Encounter

<Note>
  Requires **admin** or **practitioner** role.
</Note>

```http
POST /api/fhir/Encounter
Authorization: Bearer {token}
Content-Type: application/json
```

<ParamField body="resourceType" type="string" required>
  Must be `"Encounter"`
</ParamField>

<ParamField body="status" type="string" required>
  `planned`, `arrived`, `triaged`, `in-progress`, `onleave`, `finished`, `cancelled`, `entered-in-error`, `unknown`
</ParamField>

<ParamField body="class" type="object" required>
  Classification of patient encounter (ambulatory, emergency, inpatient)
</ParamField>

<ParamField body="type" type="array">
  Specific type of encounter
</ParamField>

<ParamField body="subject" type="object" required>
  Reference to Patient: `{"reference": "Patient/{id}"}`
</ParamField>

<ParamField body="period" type="object">
  The start and end time of the encounter
</ParamField>

<ParamField body="reasonCode" type="array">
  Coded reason for the encounter
</ParamField>

<ParamField body="participant" type="array">
  List of participants in the encounter
</ParamField>

#### Example Request

```bash cURL
curl -X POST https://api.example.com/api/fhir/Encounter \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "resourceType": "Encounter",
    "status": "finished",
    "class": {
      "system": "http://terminology.hl7.org/CodeSystem/v3-ActCode",
      "code": "AMB",
      "display": "ambulatory"
    },
    "type": [{
      "coding": [{
        "system": "http://snomed.info/sct",
        "code": "185349003",
        "display": "Encounter for check up"
      }]
    }],
    "subject": {
      "reference": "Patient/65f1234567890abcdef12345"
    },
    "period": {
      "start": "2024-03-15T09:00:00Z",
      "end": "2024-03-15T09:30:00Z"
    },
    "reasonCode": [{
      "coding": [{
        "system": "http://snomed.info/sct",
        "code": "390906007",
        "display": "Follow-up encounter"
      }]
    }]
  }'
```

### List Encounters

```http
GET /api/fhir/Encounter
Authorization: Bearer {token}
```

<ParamField query="subject" type="string">
  Filter by patient: `Patient/{id}`
</ParamField>

<ParamField query="status" type="string">
  Filter by status
</ParamField>

<ParamField query="date" type="string">
  Filter by encounter date
</ParamField>

## Encounter Classes

| Code | Display | Description |
|------|---------|-------------|
| `AMB` | ambulatory | Outpatient visit |
| `EMER` | emergency | Emergency department visit |
| `IMP` | inpatient | Inpatient admission |
| `VR` | virtual | Telemedicine encounter |

## Encounter Status Lifecycle

<Steps>
  <Step title="Planned">
    Encounter is scheduled but not yet started
  </Step>
  <Step title="Arrived">
    Patient has arrived for the encounter
  </Step>
  <Step title="In Progress">
    Encounter is currently happening
  </Step>
  <Step title="Finished">
    Encounter has been completed
  </Step>
</Steps>
